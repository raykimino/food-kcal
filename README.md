# 食物热量记录 🍎

一款运行于浏览器的**渐进式网页应用（PWA）**，无需安装、无需后端，所有数据本地存储。支持添加到手机主屏幕离线使用。

---

## 功能特性

- **饮食日历** — 月历视图，有记录的日期显示绿点，点击切换查看
- **分餐段记录** — 早餐 / 午餐 / 晚餐 / 加餐零食四个餐段独立统计
- **三大营养素** — 可选填蛋白质、碳水化合物、脂肪，当日汇总一览
- **每日目标进度条** — 自定义热量目标，超标变红、接近变橙
- **收藏夹快选** — 收藏常吃食物，一键填充名称与营养信息
- **历史记忆自动补全** — 再次输入相同食物名称自动填入上次热量
- **复制记录** — 将任意日期的全部记录一键复制到当前日期
- **快速克隆** — 单条食物旁一键复制，快速添加重复项
- **批量删除** — 多选模式批量删除食物条目
- **热量趋势图表** — 近7天 / 本周 / 本月三种视角，含总计、日均、有记录天数统计
- **完成打卡** — 打卡按钮触发撒花庆祝动画 + 手机震动反馈
- **数据导出 / 导入** — JSON 格式备份与恢复，兼容旧版数据格式
- **深色模式** — 自动跟随系统主题切换
- **PWA 支持** — 可安装到手机主屏幕，支持离线使用

## 快速开始

本项目为纯静态应用，直接用浏览器打开 `index.html` 即可，无需安装任何依赖。

```bash
# 克隆项目
git clone <仓库地址>

# 直接打开
open index.html
# 或用 VS Code Live Server / 任意本地 HTTP 服务器访问
```

**添加到手机主屏幕（iOS / Android）：**
1. 用 Safari / Chrome 打开页面
2. 点击「分享」→「添加到主屏幕」
3. 即可像原生 App 一样全屏使用，支持离线

## 数据存储

| 键名 | 内容 |
|------|------|
| `foodTrackerDataV2` | 所有饮食记录（按日期索引） |
| `foodTrackerGoal` | 每日热量目标（默认 2000 kcal） |
| `foodTrackerFavorites` | 收藏的常用食物列表 |

数据全部存储于浏览器 `localStorage`，不上传任何服务器。清除浏览器数据前建议先**导出 JSON 备份**。

### 食物条目数据结构

```json
{
  "id": 1710000000000,
  "date": "2026-03-20",
  "meal": "lunch",
  "name": "米饭",
  "calories": 232,
  "protein": 4.3,
  "carbs": 50.0,
  "fat": 0.5
}
```

> `protein` / `carbs` / `fat` 为可选字段，旧版数据不含这些字段可正常导入使用。

## 修改日志

### v2.0.0 — 2026-03-20

**新增功能：**
- **#1 复制记录**：明细卡新增「复制记录」按钮，输入源日期后一键将该日全部饮食条目复制到当前选中日期
- **#2 收藏夹快选**：表单上方增加收藏夹芯片区，填好食物后点「加入收藏」保存，点击芯片自动填充名称/热量/营养素，支持移除
- **#5 三大营养素追踪**：表单增加可折叠的蛋白质/碳水/脂肪输入区；当日摘要卡显示三项合计；食物条目行内展示宏营养素详情
- **#8 周/月统计**：趋势图表增加「近7天 / 本周 / 本月」Tab 切换；图表下方新增总热量、日均热量、有记录天数三项统计
- **#9 快速克隆**：每条食物条目旁增加复制按钮，一键在同一日期新增一条完全相同的记录
- **#12 批量删除**：明细卡新增「批量选择」模式，支持逐项勾选或全选后批量删除

**优化：**
- 历史记忆自动补全现同步填充营养素字段（如该食物有历史营养素数据）
- 收藏夹与历史记忆联动，选中收藏食物后自动展开营养素面板

**兼容性：**
- 导入旧版（v1）JSON 数据时自动补全 `meal` 字段，完全向前兼容
- `protein / carbs / fat` 为可选字段，旧数据缺失时页面正常显示，不影响任何现有功能
